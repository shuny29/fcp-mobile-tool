# Rough Cut X(iPhone版・PWA)

iPhoneのSafariだけで動く、無音自動カット + 日本語自動字幕 + 音量均一化 + 学習ツールです。
「ラフカット(仮編集)」をワンタップで自動生成し、Final Cut Proでの仕上げにそのまま
つなげられることから「Rough Cut X」と名付けました。
**すべての処理が端末内で完結**します(動画や音声をどこにも送信しません)。
生成した「無音カット済みFCPXML」と「字幕SRT」をAirDropやiCloud DriveでMacに送り、
Final Cut Proに読み込んで仕上げる、という運用を想定しています。

## 見た目について

Final Cut Proの純正インスペクタパネルを意識したダークUIにしています。

- 各ステップは折りたたみ可能な「インスペクタパネル」(タップで開閉)
- スライダーやセグメントコントロール(ピルボタン)はFCPのパラメータ調整に近い見た目
- 無音カットの結果は波形ストリップ(ミニタイムライン)でひと目で確認可能
- ヘッダーの進捗ドットで、今どのステップにいるかが分かります
- フッターに現在のバージョンを表示(更新されたか一目で分かります)

## できることと流れ(①〜⑥)

1. **動画を選ぶ**: iPhoneの写真アプリから選択すると、アップロードエリアの
   アイコンがそのままサムネイル(静止画+再生バッジ)に変わります。その後
   表示される「動画を解析」ボタンをタップして解析を開始します
   (このタップ操作でiOSの音声再生許可を確実に得ています)
2. **無音区間を検出**: 設定不要・全自動。人の声を解析し、無音区間を自動検出します
3. **区間の確認 & 字幕生成**: 残す/カットする区間をチェックボックスで調整し、
   字幕モデルと音量の最適化先を選んで生成(音量の自動均一化も同時に実行)
4. **字幕を確認・修正**: 誤認識を直し、その場で「この字幕の修正を学習させる」
5. **仕上がりを確認**: カット・音量調整・字幕を反映した状態で、書き出し前に
   そのまま再生してプレビューできます
6. **fcpxml形式で書き出し**: AirDropまたはファイル保存でMacへ

## 動画の選び方について

ファイル選択時に「写真ライブラリ」「ビデオを撮る」「ファイルを選択」の
3択が表示されますが、**「ファイルを選択」を選んでください**。iCloudに
保存されている(端末にフルサイズがダウンロードされていない)写真ライブラリの
動画は、正しく読み込めない場合があることが分かっています。「ファイルを選択」
経由であれば安定して読み込めます。iOSの標準機能の制約のため、Webアプリ側から
写真ライブラリの選択肢自体を非表示にすることはできませんが、ヒントとして
画面にも案内を表示しています。

**MacBook(デスクトップ)でファイル選択ができない場合**: 以前、上記のiOS対策として
`accept`属性を特定の形式に絞っていましたが、これがMacBookのSafari/Chromeで
ファイル選択自体を妨げてしまう副作用があったため、より汎用的な指定に戻しました
(v1.15.0で修正済み)。

## 無音区間の検出について(保守的な設定)

「明らかな無音・間だけをカットし、短い間や語尾の余韻は残す」という方針で
調整しています。Final Cut Pro側での最終調整を前提に、迷ったときは
「カットしない」側に倒すようにしています。

以前はしきい値・無音の長さ・余白をスライダーで手動調整する仕様でしたが、
現在は**完全自動**です。

- 人の声の帯域(だいたい100Hz〜4000Hz)だけを取り出して解析するため、
  空調音やヒスノイズなどを声と誤認しにくくなっています
- そのクリップ自身の音量分布から「暗騒音」と「発話」の目安を自動的に求め、
  しきい値を決定します(録音レベルや話者の声量に左右されにくい)
- **冒頭の無音は常にカット対象から除外**されます(最初の発話の前に少し
  余白を残す、一般的な編集の慣習に合わせています)

## テロップの自動生成について

このツールは「SRT字幕の読み込み」ではなく、**FCPXMLの中に直接テロップ
(Basic Titleクリップ)として埋め込む**方式を採用しています。書き出した
`_cut.fcpxml`をFCPで開くだけで、カット・音量調整・テロップがすべて
入った状態のタイムラインになり、あとはFCP側でフォントやデザインを
仕上げるだけで済みます(SRTのように別途インポートし直す手間がありません)。

### 整形ルール

- 1行あたり最大13文字
- 句読点(。、！？等)の直後を優先して改行する
- 1つのテロップは最大2行まで。収まらない場合は、文字数に応じて時間を
  按分し、複数のタイムスタンプに自動分割する
- 位置は「画面中央より少し下」をデフォルトにしています(FCPの
  インスペクタでいつでも位置・フォント・サイズを調整できます)

### 技術的な補足・制約

- テロップはFCP標準の「Basic Title」という組み込みタイトルを使って
  生成しています。書き出したFCPXMLをFCPで開くと、カット済みの映像の上に
  テロップのクリップが重なった状態で配置されます。
- **この機能はSRT字幕の書き出しに比べて技術的に複雑**で、実機のFCPで
  網羅的に検証できているわけではありません。初回はインポート結果を
  必ず確認してください。万一テロップの位置や表示がおかしい場合は、
  FCPのインスペクタ(Command+4)から個別に調整できます。
- 念のため、従来通りのSRT字幕ファイルも一緒に書き出しています
  (テロップがうまく読み込めない場合の予備・参考用です)。
- **話者(人物)ごとにテロップを分ける機能はありません。** このツールは
  「誰が話しているか」を判定する話者分離機能を持たないため、話者が
  切り替わる場面でテロップが混在しないことは保証できません。無音区間の
  検出結果に沿ってテロップが区切られるため、間があれば自然に区切られますが、
  会話が途切れず続く場面では区別できない点をご了承ください。

## 音量の自動均一化について(無音検出の「後」に実行)

以前は「無音検出の前に音量を均す」設計にしていましたが、これが原因で
**録音中に声量が変化すると、それ以降の発話がすべて無音と誤判定されて
消えてしまう重大な不具合**が見つかりました。音量調整とカット判定を
同時に行うと、片方の不完全さがもう片方に影響してしまうためです。

そのため現在は、**無音検出が完全に確定した「後」に、区間ごとの音量を
別途計算する**設計に変更しています(検出結果には一切影響しません)。

- 検出された各発話区間の平均音量を測定し、区間同士の「中央値」に
  揃えるための調整量(dB)を計算します
- 人の声を一律同じ音量に均すのが目的のため、SNS向け等のプリセット選択は
  廃止し、常に同じ基準で調整します
- 実際の調整量が、そのままFCPXMLの非破壊ゲイン(`<adjust-volume>`)として
  書き出されます。区間一覧に「+9.0dB」のように表示されます

## 無音検出の精度について

以前は「一度無音と判定されたら、発話に戻るにはしきい値を数dB上回る必要がある」
ヒステリシス(幅のあるしきい値)を設けていましたが、これが原因で**録音中の
声量変化や語尾の余韻の場面で、一度無音と判定されると二度と発話と判定されなく
なり、その後の発言がまるごとカットされてしまう重大な不具合**がありました。

現在は「無音に入る」しきい値と「発話に戻る」しきい値を**同じ**にし、この
トラップを解消しています。短い揺らぎでの誤カットは、ヒステリシスではなく
「立ち上がりは速く・減衰はゆっくり」な包絡線(エンベロープ)の平滑化と、
最小無音長(約800ms)の判定で防いでいます。また、無音/発話の目安となる
音量分布も、クリップ内で一番大きい声だけに引っ張られないよう、控えめな
パーセンタイルを採用しています。

## 先に知っておいてほしいこと(制約・注意点)

- **モデルの初回ダウンロードが必要**です。「base」モデルで約145MB、Wi-Fi推奨です。
  一度ダウンロードすればブラウザにキャッシュされ、以降はオフラインで動きます。
- 端末内処理のため、PC版Whisper(large等)と比べると**認識精度はやや落ちます**。
  専門用語や固有名詞は「学習」機能で徐々に賢くなっていきます。
- **動画の音声読み込みについて**: iPhoneで撮影した.mov/.mp4は、Safariの高速デコード
  (`decodeAudioData`)では失敗・またはタイムアウトすることがあります。その場合、
  自動的に「動画を実際に(小さなプレビューとして画面に表示しながら)再生して
  音声を捕まえる」方式に切り替わります。この方式は**動画の長さとほぼ同じだけの
  実時間がかかります**(例: 6秒の動画なら6〜10秒程度)。画面右下(または各ステップの
  上)に小さなプレビュー動画が表示され、「音声を解析中」の表示とともに進捗が
  出ます。数分経っても終わらない場合は不具合(ウォッチドッグにより自動的に
  エラー表示されるようになっています)なので、もう一度お試しください。
- 長い動画(10分超など)でのメモリ使用量・処理時間は機種・iOSバージョンによって
  挙動が変わる可能性があるため、実機で試しながら調整してください。
- BGMや環境音が大きいと無音判定の精度が落ちるのはPC版と同様です。

## セットアップ(初回のみ・GitHub Pagesを使う方法)

PWAとして正しく動作(ホーム画面追加・オフラインキャッシュ)させるには、**HTTPSでの配信**
が必要です。一番簡単で無料な方法はGitHub Pagesです。

1. GitHubでアカウントを作成(すでにお持ちなら不要)
2. 新しいリポジトリを作成(例: `fcp-mobile-tool`)
3. このフォルダの中身一式(`index.html`や`js/`フォルダなど)をリポジトリにアップロード
   - GitHubのWebサイト上で「Add file → Upload files」からドラッグ&ドロップでもOKです
4. リポジトリの `Settings → Pages` を開き、`Source` を `main` ブランチ / `/ (root)` に設定して保存
5. 数分待つと `https://ユーザー名.github.io/fcp-mobile-tool/` のようなURLが発行されます

## iPhoneでの使い方

1. 発行されたURLをiPhoneのSafariで開く
2. 共有ボタン(下部の四角に上矢印のアイコン) → 「ホーム画面に追加」
3. ホーム画面のアイコンから起動(アプリのように全画面で開きます)
4. 動画を選択 → 無音区間検出 → 区間確認 → 字幕生成 → 修正 → 学習 → Macへ送る

## なぜこの方式(Safari + リアルタイムキャプチャ)を採っているか

Safari(WebKit)は、動画コンテナ(.mov/.mp4)から音声だけを取り出す
`decodeAudioData`が不安定という制約があります。これを回避するため、本アプリは
「動画を実際に再生し、その音声をWeb Audio APIで捕まえる」方式を採用しています
──これはブラウザだけで完結させる場合の、現実的に最も互換性の高い方法です。

ただし欠点として、**処理時間が動画の長さにほぼ比例します**(倍速化すると音声の
ピッチが変わってしまい、字幕生成や音量測定の精度が落ちるため、等速のままに
しています)。

より高速な代替案として、ffmpeg.wasm等のWASM版動画デコーダーを組み込み、
再生を介さず直接コンテナから音声データを取り出す方法もあります。この方法なら
理論上は動画の長さに関係なく高速に処理できますが、初回に数十MBの追加
ダウンロードが必要になり、実装もより複雑になります。現状の速度で困っている
場合はお知らせください、この方式への切り替えも可能です。

## Final Cut Pro側の手順(Mac)

1. AirDropまたはiCloud Drive経由で受け取った `xxx_cut.fcpxml` を
   `File > Import > XML...` で読み込む(無音カット・音量調整・テロップが
   すべて入ったシーケンスが作成されます)
2. あとはFCP側で細かいカット・微調整・テロップのフォントやデザインを
   仕上げてください(テロップのクリップを選択し、Command+4でインスペクタを
   開くとフォント・サイズ・色・位置などを変更できます)
3. 万一テロップがうまく読み込まれない場合は、同時に書き出される
   `xxx_subtitles.srt` を予備として `File > Import > Captions...` で
   読み込んでください

## 学習機能について

- **誤認識辞書**: ステップ4で字幕を修正し、「この字幕の修正を学習させる」ボタンを
  押すと、「誤認識 → 正しい語」のペアを自動抽出してブラウザのlocalStorageに保存
  します。次回の文字起こし時に自動で置換されます。
- 無音区間の検出は完全自動化されたため、学習させる項目は現在このみです。
- 学習データは端末(ブラウザ)ごとに保存されます。Safariの「Webサイトデータを消去」を
  行うとリセットされる点にご注意ください。

## ファイル構成

```
fcp_mobile_pwa/
├── index.html            # 画面本体
├── manifest.webmanifest  # ホーム画面追加用の設定
├── sw.js                 # オフラインキャッシュ(Service Worker)
├── css/style.css
├── js/
│   ├── app.js             # 画面とロジックを繋ぐメイン処理
│   ├── silence.js         # 無音区間検出(Web Audio API)
│   ├── asr.js              # 端末内Whisper文字起こし(Transformers.js)
│   ├── loudness.js         # (現在未使用/旧方式のラウドネス測定。音量調整はsilence.js内で実施)
│   ├── export.js           # FCPXML/SRT生成・タイムライン再マッピング
│   └── learning.js         # 学習機能(localStorage)
└── icons/                 # ホーム画面アイコン
```

## バージョン表記について

フッターに表示される `Rough Cut X vX.X.X` は、更新が実際に反映されたかを
確認するための目印です。コードを更新する際は、`index.html` フッター内の
バージョン番号と、`sw.js` の `CACHE_NAME` を両方1つずつ上げる運用にして
ください(数字を合わせる必要はありませんが、両方を必ず更新してください)。

## トラブルシューティング

- **長い動画で、ほとんどの区間が無音として誤ってカットされる**
  フォールバック方式(動画を実際に再生して音声を取得する方式)で、動画要素を
  ミュートしたまま音声を取得すると、ブラウザによってはWeb Audio側に渡る
  音声データ自体が極端に減衰してしまう不具合がありました。実際に再生が
  始まった直後にミュートを解除する(スピーカーから聞こえないのは別の仕組みで
  担保)よう修正しています(v1.21.0)。②の「詳細情報」に表示される
  「取得方式」が「フォールバック」になっている場合に起きやすい症状でした。

- **「動画の読み込みがタイムアウトしました」が、動画の長さに関わらず必ず約25秒で出る**
  内部の「起動時の保険用タイマー」を、通常の(動画の長さに応じた)タイマーに
  切り替えた後も解除し忘れており、どんな動画でも一律25秒で強制終了して
  しまう不具合がありました(v1.20.0で修正済み)。短い動画では気づきにくく、
  1分前後の動画で顕在化していました。
- **「動画の読み込みがタイムアウトしました」/「音声データを取得できませんでした」
  が繰り返し出る**
  iOSでは、音声再生の許可(AudioContextの有効化)は「直接的なタップ操作の中」で
  行わないと確実に効かないことが分かりました。ファイル選択(写真アプリの
  ピッカー操作)そのものは、この目的では有効なタップとして扱われない端末が
  あるようです。v1.10.0からは、ファイル選択後に表示される「動画を解析」
  ボタンのタップ自体で音声を有効化するように変更し、解決を図っています。
  iOSでは「ユーザー操作の有効期限」が短く、ファイル選択から実際の音声取得
  までの間に非同期処理を挟むと、その期限が切れてAudioContextの再開が効かなく
  なることがありました。ファイル選択の直後、他の処理より前にAudioContextを
  確保しておく対策を行っています(v1.8.0で修正済み)。それでも起きる場合は、
  一度Safariを完全に終了して開き直してからお試しください。
- **動画の読み込み/解析がずっと終わらない**
  動画のメタデータ取得自体が固まってしまうケースへの保険(25秒でタイムアウト)を
  追加しています。それでも起きる場合は、動画のファイル形式やサイズに起因する
  可能性があるため、別の動画で試すか、iPhone標準のカメラで撮影したものを
  そのまま(編集アプリを介さず)お試しください。
- **アップロードエリアのアイコンが、サムネイル表示後も二重に見える**
  表示の切り替え漏れを防ぐ対策(インラインstyleでの二重制御)を行っています。
  それでも起きる場合は一度ページを再読み込みしてお試しください。

- **⑤のプレビューが再生されない/音が出ない**
  「この内容で書き出しへ進む」の前段階で、動画の読み込み(メタデータ取得)を
  待ってから再生位置を合わせる仕組みにしています。それでも無音・無反応の場合は、
  一度ステップ4に戻って「サンプルを再生して確認する」を押し直してください。
  音が出ない場合は端末の消音スイッチ・音量をご確認ください(音量調整はブラウザの
  音声グラフを経由するため、消音状態だと一切聞こえません)。

- **「音声の読み込みに失敗しました」と出る**
  通常は自動でリアルタイムキャプチャ方式にフォールバックするため、この画面が
  出ること自体が減っているはずです。それでも出る場合は、動画自体が壊れている
  か、極端に特殊なコーデックの可能性があります。別の動画で試すか、一度
  「写真」アプリで別形式に書き出してからお試しください。
- **アップロードしたのに更新が反映されない**
  以前のバージョンにService Workerのキャッシュの不具合があり、更新してもiPhone側に
  古い画面が残り続けることがありました(v2で修正済み)。今回だけ、一度だけ以下の
  手順でキャッシュを消してください(これ以降は自動で更新されます):
  1. iPhoneの「設定」アプリ → 「Safari」→「詳細」→「Webサイトデータ」
  2. アップロード先のドメイン(例: `ユーザー名.github.io`)を検索して削除
  3. ホーム画面のアイコンを一度削除し、Safariでサイトを開き直してから
     「ホーム画面に追加」をやり直す
  - 今後コードを更新する場合は、`sw.js` 内の `CACHE_NAME` の数字を1つ上げてから
    アップロードしてください(例: `-v2` → `-v3`)。これを忘れると、また同じ現象が
    起きます。
  - **v1.18.0から**: 新しいバージョンのService Workerが有効化されたことを検知したら
    自動的にページを1回だけ再読み込みする仕組みを追加しました。ただし、
    「開きっぱなしのタブ・PWA」が更新を検知するまでには多少時間がかかることが
    あります。すぐに確認したい場合は、一度アプリを完全に閉じて(ホーム画面の
    PWAならアプリスイッチャーからスワイプで終了)開き直してください。
  - **コードが正しいのに挙動が変わらない場合の切り分け方**: Safariの
    プライベートブラウズウィンドウで開いてみてください。プライベートブラウズは
    キャッシュ・Service Workerの影響を受けないため、そちらで正しく動作すれば
    原因はキャッシュ、動作しなければコード側の問題と判断できます。
- **「無音区間を検出」しても区間が0件になる/字幕生成で「カットする区間しかありません」と出る**
  リアルタイムキャプチャ方式(動画再生による音声取得)で、AudioContextが
  サスペンド状態のまま音声を1サンプルも取得できていない場合に起きる不具合でした
  (v3で修正済み)。それでも発生する場合は、動画選択の直後にすぐ処理が始まる
  ようにしているため、他の操作を挟まずお試しください。
- **「動画を再生しながら音声を解析中...」のまま進まない**
  この表示はフォールバック処理中で、動画の長さぶん実時間がかかるのが仕様です
  (1分の動画なら約1分)。パーセント表示が増えていれば正常に進行しています。
  増えない場合は、Safariがバックグラウンドでの動画再生を止めている可能性が
  あるため、アプリを開いた状態(画面ロックなし)でお待ちください。
- **文字起こしがずっと終わらない/固まる**
  長い動画・多い発話区間だと端末内処理はかなり時間がかかります。まずは短い動画
  (1〜2分)で試すことをおすすめします。速度を優先するなら「軽量・高速(tiny)」
  モデルに切り替えてください。
- **FCPXMLの読み込みに失敗する**
  `js/export.js` 内の `version="1.10"` を、お使いのFCPが対応するバージョンに
  変更してみてください。
- **オフラインで開けない**
  初回は必ずオンラインでアクセスしてService Workerとモデルをキャッシュさせてください。
  GitHub Pages(HTTPS)経由でない場合、Service Workerが登録されずオフライン非対応になります。

## PC版(Final Cut Pro用フル機能版)との関係

同じ考え方(FCPXMLでカット、SRTで字幕、学習機能)を、より高精度なPC版(Python+Whisper大型
モデル)としても用意しています。撮って出しの粗編集をiPhoneで素早く行い、仕上げをMacの
Final Cut Proで、という分担を想定した設計です。
